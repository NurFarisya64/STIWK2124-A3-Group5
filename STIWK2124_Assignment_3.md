Evaluate design, functionality \&

**Scenario**

Evaluate and harden your ARL application: add tests, verify security behavior, measure basic performance,   
and produce a deployable build (local production build acceptable; Docker/CI optional).

**Technical Requirements**

1.

Testing: ≥3 backend JUnit tests (service/controller with success \& failure paths) and ≥1 Angular unit

test; optional E2E smoke.

2.

Security \& Robustness: Demonstrate Basic Auth on Write Endpoints; input validation on client/server;   
show handled 400/404; briefly discuss HTTPS \& headers in reflection.

3.

Deployment Artifact: Production build (min) or optional Dockerfiles/docker-compose or GitHub Actions   
CI.

4.

Reflection (800

--1000 words): design trade‑offs, functionality/UX, simple performance measurement,

security risks \& mitigations, and next steps.

•

GitHub repo links (backend \& frontend) with tests and build scripts.

Test evidence (screenshots or CI logs).

•

Deployment artifact(s) and run instructions.

•

Reflection report (PDF/Doc).
